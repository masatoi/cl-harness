# Phase 5.1 benchmark delta

**Date**: 2026-05-05 (same day, after Phase 5.0)
**Model**: `llama-3.3-70b-versatile` via Groq
**Run-limits**: max-turns 20, temperature 0.0, max-tokens 2048
**Code**: cl-harness Phase 5.1 (load-failure prompt enhancement +
tokens in transcripts + 4 new fixtures)

## What changed since Phase 5.0

- `verify-detail-prose` now hoists a one-line "Likely cause:" diagnosis
  above the verbose load-failure dump, picked from `+load-failure-markers+`
  (`no symbol named`, `Package error`, `undefined function`, etc.). Targets
  Phase 5.0's max-turns failure on 003-missing-import, where the LLM kept
  reading src/ files instead of the test's `:import-from` clause.
- `:llm-response` transcript events now include the per-turn `tokens`
  count, so post-mortem analysis can recover token usage.
- 4 new fixtures: 004-defmethod-specializer / 005-defclass-slot /
  006-format-arity / 007-error-signal.

## 003-missing-import re-run (target of the prompt change)

| Run | Condition | Status (5.0) | Status (5.1) | Turns (5.0 → 5.1) | Patches (5.0 → 5.1) |
| --- | --------- | ------------ | ------------ | ------------------ | -------------------- |
| 003-missing-import | `generic-mcp`    | `max-turns` | **`passed`** | 20 → **4**  | 8/13 → **2/2** |
| 003-missing-import | `runtime-native` | `max-turns` | `max-turns`  | 20 → 20      | 5/6 → 3/?  |

The "Likely cause:" pointer collapsed `generic-mcp` from 20-turn failure
to a 4-turn fix with no wasted patch attempts. `runtime-native` remained
stuck at max-turns; its system prompt's emphasis on
probe-then-patch with `repl-eval` / `inspect-object` appears to dilute
the LLM's focus on the import line specifically. Worth investigating in
Phase 5.2.

## Limitations and Phase 5.2 work

- **Single trial.** Repeat runs (n=3+) needed for variance bands.
- **`runtime-native` advantage still not visible.** Same 75% as
  `generic-mcp` in 5.0 numbers (when generic-mcp passed 3/4); on this
  small + simple-bug suite, REPL probe doesn't yet pay off. Phase 5.2
  should add probe-friendly tasks (e.g. type confusion that benefits
  from `(repl-eval (foo bad-input))` reproduction; macroexpansion bugs).
- **`file-only` was not re-run.** Phase 5.0 numbers (0/4) stand; the
  prompt change does not affect file-only's full-file-rewrite weakness.
- **003 × runtime-native** still max-turns. Worth a focused diagnosis:
  the runtime-native workflow prompt may need to explicitly call out
  `:import-from` typos as a special case where probing isn't useful.
- **Token usage column** is now reliable (recorded in transcripts since
  this commit).
- **Total tasks: 8 of the PRD §17 target of 10+.** Need 2 more in
  Phase 5.2 (e.g. macroexpansion bug and condition-handler bug).
